Submodules¶

class saga.source.extractor.mindsight.logic.LogicExtractor(base_url: str, token: str, limit: int = 300, tries: int = 1)¶

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract data from the “logic/” endpoint of the API.

This endpoint contains the data of all logic tests performed by the registered users. One key factor to take into account is that this endpoint has no query string or parameter to filter per date, forcing a full data extract every day to detect updates.

Parameters

  • base_url (str) – Base API url.

  • token (str) – API auth token.

  • limit (int, optional) – Number of results per page. The API max is 300. Defaults to 300.

  • tries (int, optional) – Number of retries if the request fails. Defaults to 1.

extract()¶

Yields the API response for this endpoint. In this case, yields a dictionary containing a list of results, where each result is an user logic test report.

Yields

dict – Json containing the API results
