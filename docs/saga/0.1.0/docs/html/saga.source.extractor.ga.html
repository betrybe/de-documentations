class saga.source.extractor.ga.anonymous_id_extractor.AnonymousIDExtractor(ga_client: object, view_id: str, start_date='2005-01-01', end_date='yesterday', limit: int = 10000)¶

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract user traffic data from the GA API. Every user on the platform holds an ID, called the anonymous ID. This extractor gets all the data related to user traffic based on date intervals. The most common extraction is a daily one, but this class gives the option to extract from any period of time if necessary, respecting the API limits, such as the minimum start date of “2005-01-01”.

Parameters

  • ga_client (object) – GA API object already intialized with credentials.

  • view_id (str) – GA view ID to extract the data. Must be a string.

  • start_date (str, optional) – Initial extraction date. Defaults to “2005-01-01”.

  • end_date (str, optional) – End extraction date. Defaults to “yesterday”.

  • limit (int, optional) – API page limit. Defaults to 10000.

extract()¶

Yields the API response. In this case, get all the user traffic data.

Yields

dict – Json containing the API data.

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract the enrollment count per day.

Number of people who enrolled in the marketing form. To extract this, the metric “ga:goal7Completions” is used. This metric represents finalized enrollments. For the dimensions, the following are used: “ga:source”, “ga:medium”, “ga:campaign”, “ga:channelGrouping” and “ga:date”.

The most common extraction is a daily one, but this class gives the option to extract from any period of time if necessary, respecting the API limits, such as the minimum start date of “2005-01-01”.
