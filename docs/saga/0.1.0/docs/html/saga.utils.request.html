class saga.utils.request.api.Api¶

Bases: object

Api class, wrapper around requests module.

static get(url: str, path: str = '', payload: Optional[dict] = None, tries: int = 1, pagination: bool = False, page_keys: tuple = ('next_page'), **extra_args: dict)¶

Wrapper around request.get() method. Used to encapsulate some uses cases, such as request retry and pagination.

Parameters

  • url (str) – Target base url to get data from

  • path (str, optional) – Base url path. Defaults to “”.

  • payload (str, optional) – Extra data contained in the full url, like api parameters that need to be encoded on the url. All data contained here will be encoded on the url, after the desired path, using the format “url/path?payload”. Defaults to “”.

  • tries (int, optional) – The number of request retries. If the request fails, it will retry until it reaches the number o tries specified here. Defaults to 1.

  • pagination (bool, optional) – If the api uses pagination, set this flag to True. Defaults to False.

  • page_key (tuple, optional) – The pagination keys contained in the json response of the api. Defaults to (“next_page”,).

  • **extra_args (dict) – Extra get arguments passed to requests.get method

Yields

dict – Json returned from the API

class saga.utils.request.GA¶

Bases: object

GA class, wrapper around the google analytics API module. Used to paginate through results.

static paginate(ga_client: object, request_body_maker: Callable[], dict])¶

Paginate through GA analytics API responses

Parameters

  • ga_client (object) – GA API object, already intialized with credentials and ready to be used.

  • request_body_maker (Callable[[], dict]) – Method to create a new body with the necessary keys for the API calls. This method is necessary because the body needs to be reseted every iteration of the pagination, thus removing the “pageToken” key and cleaning the dict.

Yields

dict – Json containing the response
