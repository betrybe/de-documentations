class saga.source.extractor.hubspot.companies.CompaniesExtractor(base_url: str, token: str, start_date: Optional[datetime.date] = None, end_date: Optional[datetime.date] = None, sort_key: str = 'hs_lastmodifieddate', archived: bool = False, limit: int = 100, tries: int = 1)¶

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract data from the “companies/” endpoint of the API. This endpoint contains the data of all companies collected by the CRM API. Since this endpoint supports filtering, it’s possible to use the field “hs_lastmodifieddate” to get only the most recent events.

Parameters

  • base_url (str) – Base API url.

  • token (str) – API auth token.

  • start_date (date) – Start period to query results (closed interval). Defaults to “1970-01-01”.

  • end_date (date) – End period to query results (open interval). Defaults to current day.

  • sort_key (str) – Property name to order results. Defaults to “hs_lastmodifieddate”.

  • archived (bool) – Whether to query archived deals or not. Defaults to False.

  • limit (int, optional) – Number of results per page. The API max is 100, if a

  • value is provided, a 400 status code is returned. Because of this, (higher) –

  • limit is capped at 100, even if more than 100 is passed. (this) – Defaults to 300.

  • tries (int, optional) – Number of retries if the request fails. Defaults to 1.

extract()¶

Yields the API response for this endpoint. In this case, yields a dictionary containing a list of results, where each result is company data. First, makes a GET request to retrieve all possible properties of the “companies” object and than uses them to retrieve the companies data.

Yields

dict – Json containing the API results

class saga.source.extractor.hubspot.contact_identity_profiles.ContactIdentityProfilesExtractor(base_url: str, token: str, limit: int = 250, tries: int = 1)¶

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract data from the “contacts/v1/lists/all/contacts/all” endpoint of the API. This endpoint contains the data of all contacts identity profiles collected by the CRM API. Since this endpoint does not support filtering, every day all data is extracted for later use.

Parameters

  • base_url (str) – Base API url.

  • token (str) – API auth token.

  • limit (int, optional) – Number of results per page. The API max is 250, if a higher value is provided, the API simply ignores this limit and uses 250 instead. Defaults to 250.

  • tries (int, optional) – Number of retries if the request fails. Defaults to 1.

extract()¶

Yields the API response for this endpoint. In this case, yields a dictionary containing a list of results, where each result is the contact details with it’s identity profiles. Makes a GET request with pagination to get all the data. The pagination works using the keys “has-more” and “vid-offset” yielded in the API response, where the “has-more” key indicates if there is a next page to be requested and the “vid-offset” key is used on the next request to get the next result page.

Yields

dict – Json containing the API results

class saga.source.extractor.hubspot.contact_lists.ContactListsExtractor(base_url: str, token: str, limit: int = 250, tries: int = 1)¶

Bases: saga.source.extractor.extractor.Extractor

Holds the logic to extract data from the “contacts/v1/lists/” endpoint of the API. This endpoint contains the data of all contact lists collected by the CRM API. Since this endpoint does not support filtering, every day all data is extracted for later use. To filter only new records from this endpoint, it’s possible to use the fields “updatedAt”, “metaData.lastProcessingStateChangeAt” and “metaData.lastSizeChangeAt”.

Parameters

  • base_url (str) – Base API url.

  • token (str) – API auth token.

  • limit (int, optional) – Number of results per page. The API max is 250, if a higher value is provided, the API simply ignores this limit and uses 250 instead. Defaults to 250.

  • tries (int, optional) – Number of retries if the request fails. Defaults to 1.

extract()¶

Yields the API response for this endpoint. In this case, yields a dictionary containing a list of results, where each result is contact list. Makes a GET request with pagination to get all the data. The pagination works using the keys “has-more” and “offset” yielded in the API response, where the “has-more” key indicates if there is a next page to be requested and the “offset” key is used on the next request to get the next result page.
